# Product

## Register

brand

## Users

Visitors who might want to collaborate or use Anggie's skills — small business owners, fellow learners, anyone with a simple web project idea. No specific persona; the site speaks to anyone curious enough to read it.

## Product Purpose

Showcase what Anggie builds and how Anggie works, in a way that invites collaboration. The site itself is proof of capability — if the portfolio looks good and works well, it answers the question "can this person build for me?"

## Brand Personality

Honest, fast learner, curious. The voice is someone who doesn't pretend to be senior but demonstrates real capability through shipped work. No chest-beating, no false humility — just clear evidence of learning speed and output quality.

## Anti-references

- Typical dev portfolio with GitHub contribution graphs, star counts, tech-stack laundry lists
- Corporate SaaS landing page with hero-metric gradients and identical card grids
- Generic "creative agency" portfolio with oversized hero text and stock imagery
- Anything that hides the human behind buzzwords

## Design Principles

1. **Show, don't claim** — Every section proves capability through work, not assertions
2. **Lab notebook honesty** — Failures and lessons are features, not things to hide
3. **Simple works** — Ship the small, clear, useful thing over the over-engineered system
4. **Building in public** — The process is visible and inviting, not a black box

## Accessibility & Inclusion

Existing foundations: skip links, ARIA labels, focus-visible states, keyboard navigation, reduced-motion support, high-contrast mode. Continue building on these; no specific WCAG target beyond what's already in place.
